# Glossary

| Term | Definition |
|------|------------|
| **Admin** | A user role with full CRUD access to products, categories, offers, and user management. |
| **Banner** | A promotional image displayed as a slider on the home screen. |
| **Category** | A grouping of products with Arabic and English names and an image. |
| **Customer** | A user role with read-only access to products, categories, and offers. |
| **DTO** | Data Transfer Object - an object that carries data between processes, used for serialization/deserialization. |
| **Entity** | A domain object representing core business data, immutable and framework-agnostic. |
| **Feature** | A self-contained module grouping UI, state, and logic for a specific functionality. |
| **Firestore** | A NoSQL document database by Firebase used for real-time data storage. |
| **Localization** | The process of adapting the application for Arabic (RTL) and English (LTR) languages. |
| **Offer** | A promotional campaign with a title, description, image, validity dates, and associated products. |
| **Offline-First** | A strategy where the app functions without connectivity and syncs when online. |
| **Product** | An item in the catalog with price, weight, and localized name/description. |
| **Repository** | A design pattern that mediates between domain and data mapping layers. |
| **Riverpod** | A state management library for Flutter with compile-time safety and dependency injection. |
| **RTL** | Right-to-Left script direction used for Arabic language layouts. |
| **Use Case** | A single unit of business logic in the domain layer. |
| **Weight Unit** | A lookup value defining measurement units (e.g., kg, g, lb). |
